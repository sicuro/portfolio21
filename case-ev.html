Problema

A Estante Virtual é uma plataforma de marketplace onde seus principais clientes são sebos. Um dos maiores problemas da Estante Virtual, está na parte onde onde o cliente (sebo), não consegue cadastrar de forma rápida todo o seu acervo. Com isso, somente cerva de 40% do acervo é disponibilizado na Estante Virtual.

Um dos principais fatores para isso acontecer, é a questão de tempo gasto para o cadastro na plataforma. Podendo ser cadastrado via computador ou pelo site em sua versão responsiva, que não era muito otimizada para dispositivos mobile.

Metas

  • Aumento de disponibilização de acervo
  • Melhorar experiência de uso no cadastro de novos livros
  • Aumentar engajamento do lojista
  • Facilitar administração de acervo

Processo de pesquisa
com o usuário

Para identificar os maiores problemas até chegar a necessidade do desenvolvimento de um aplicativo para cadastro de acervo para os nossos lojistas, fizemos primeiramente um mapeamento de personas dos nossos usuários. Desde o usuário que é um vendedor ocasional até o super livreiro que possuía um acervo gigantesco, porém, não possuía força tecnológica para fazer uma conexão via API, por exemplo.

Com o mapeamento realizado, selecionamos alguns lojistas para realizarmos uma pesquisa sobre como era o dia-a-dia dele e como era a sua interação com a Estante Virtual, para conseguirmos entender suas diferentes perspectivas de jornadas.
